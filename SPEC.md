## 1. 项目概述

### 1.1 项目目标

从零手动实现完整的 RAG（检索增强生成）链路，不依赖 LangChain、LlamaIndex 等现成框架，帮助理解 RAG 的核心原理。

### 1.2 核心价值

| 特性 | 说明 |
|------|------|
| **零框架依赖** | 所有组件手工实现，便于学习原理 |
| **中文优化** | 使用阿里云 DashScope（通义千问）中文模型 |
| **轻量级** | 核心代码约 2000 行，适合教学和快速原型 |
| **可扩展** | 模块化设计，向量数据库可切换 Memory/FAISS |
| **多用户** | 支持用户注册/登录、权限分级、对话历史持久化 |

### 1.3 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 语言 | Python 3.13+ | 最新 Python 版本 |
| LLM | DashScope qwen-plus | 通义千问大模型 |
| Embedding | text-embedding-v2 | 1536 维中文优化向量 |
| 向量存储 | Memory (numpy) + FAISS | 两种后端可切换 |
| BM25 | jieba 分词 + 自实现 | 关键词检索 |
| 混合检索 | 自实现 HybridRetriever | 向量 + BM25 互补 |
| 前端 | Gradio 6 | Web 界面 |
| 分词 | jieba | 中文分词 |
| 用户管理 | 本地 JSON 文件 | users.json |
| 权限系统 | 4 级（public/internal/secret/top_secret） | 角色权限可配置 |
| 对话历史 | 本地 JSON 文件 | 每用户独立存储 |

## 3. 功能模块详解

### 3.1 文档加载 (src/document_loader.py)

| 功能 | 说明 |
|------|------|
| 支持格式 | TXT, DOCX, XLSX, PPTX, PDF, HTML |
| 分块策略 | 固定字符数 + 重叠 + 句子边界切割 |
| 默认参数 | chunk_size=500, overlap=100 |
| 元数据 | 记录文件名、修改时间、大小、文档级别 |
| 文档级别 | 每个文件夹名对应一个权限级别 (public/internal/secret/top_secret) |

**文档级别规则**：
- `documents/public/` → public 级别（所有登录用户）
- `documents/internal/` → internal 级别（user 及以上）
- `documents/secret/` → secret 级别（admin 及以上）
- `documents/top_secret/` → top_secret 级别（仅 admin）

### 3.2 文本嵌入 (src/embedding.py)

| 功能 | 说明 |
|------|------|
| 模型 | text-embedding-v2 (1536 维) |
| API | DashScope TextEmbedding |
| 分批 | 每批最多 10 条（API 限制） |
| 错误处理 | 失败时抛出 RuntimeError |

### 3.3 向量数据库 (src/vector_db/)

这是 v0.5+ 的核心重构：从单一的 `vector_store.py` 拆分为多模块向量数据库。

#### 3.3.1 base.py — 抽象基类

定义 `VectorDBBase`：
- `build(chunks, vectors, sources)`: 构建索引
- `add(chunks, vectors, sources)`: 增量添加
- `search(query_vector, top_k)`: 检索
- `save(dir_path, metadata)`: 持久化
- `load(dir_path)`: 从磁盘加载

#### 3.3.2 memory_db.py — 内存实现（默认）

| 功能 | 说明 |
|------|------|
| 存储方式 | 内存 (numpy array) |
| 相似度算法 | 余弦相似度 |
| 持久化 | vectors.npy + chunks.json + sources.json + metadata.json |
| 0维数组修复 | 使用 flatten() 替代 squeeze()，避免单文档时的 IndexError |

#### 3.3.3 faiss_db.py — FAISS 实现（可选）

| 功能 | 说明 |
|------|------|
| 存储方式 | FAISS IndexFlatL2 |
| 优势 | 大规模数据 (>10000 块) 性能显著优于 numpy |
| 持久化 | FAISS index.bin + JSON 元数据 |

#### 3.3.4 bm25_retriever.py — BM25 关键词检索

| 功能 | 说明 |
|------|------|
| 实现原理 | OKAPI BM25 |
| 分词 | jieba 中文分词 |
| **去重机制** | 增量添加时自动检测重复文档并跳过 |
| 词频管理 | 重新计算 term_doc_freq，避免重复文档导致 IDF 偏差 |
| sources 对齐 | 自动填充默认值，确保与文档数量匹配 |
| 持久化 | bm25_index.json |

#### 3.3.5 hybrid_retriever.py — 混合检索

| 功能 | 说明 |
|------|------|
| 检索组合 | vector_db + bm25_retriever |
| 融合算法 | 加权 (weight) 或 RRF |
| 空检查 | 任一数据库为空时安全回退 |

#### 3.3.6 __init__.py — 工厂模式

`VectorDBFactory.create(db_type)`: 根据配置创建向量数据库实例。

### 3.4 检索器 (src/retrieval.py)

| 功能 | 说明 |
|------|------|
| Query 改写 | 用 LLM 优化模糊/简短问题 |
| 混合检索 | 向量 + BM25，权重可配置 |
| Rerank | 用 LLM 对候选结果重新排序 |
| **权限过滤** | 按用户角色过滤文档级别 |
| 默认 fetch_k | top_k × rerank_factor |

### 3.5 RAG 流水线 (src/rag_pipeline.py)

| 功能 | 说明 |
|------|------|
| 知识库构建 | 加载文档 → 分块 → Embedding → 存储 |
| 知识库加载 | 从磁盘加载已有向量库 |
| **元数据管理** | `_doc_metadata` 始终为 dict，避免 None 操作 |
| 增量更新 | 检测文档变更，仅更新变更部分 |
| 变更检测 | 对比文件 mtime 和 size |
| **用户角色问答** | `answer()` 接受 `user_role` 参数，传递给检索器 |
| 回答生成 | Prompt 构建 + LLM 调用 |

### 3.6 用户管理 (src/user_manager.py)

| 功能 | 说明 |
|------|------|
| 用户注册 | 用户名 + 密码（hashlib + salt） |
| 用户登录 | 验证用户凭证 |
| 用户列表 | 获取所有用户信息 |
| 默认 Admin | 系统启动时自动创建默认管理员 |
| 存储 | data/users.json |

### 3.7 认证与授权 (src/auth.py)

| 功能 | 说明 |
|------|------|
| 密码哈希 | hashlib.pbkdf2_hmac + random salt |
| Session 管理 | 简单 token-based 认证 |
| 角色分配 | 新用户默认 role="user"，admin 手动创建 |

### 3.8 权限系统 (src/permission.py)

| 功能 | 说明 |
|------|------|
| 权限级别 | public → internal → secret → top_secret（4 级） |
| 角色定义 | admin / user / guest |
| 可配置 | 通过 data/roles.json 自定义角色权限 |
| 文档级过滤 | 检索后按用户角色过滤文档来源 |
| 来源格式 | `filename|level`（如 `secret_doc.txt|secret`） |

### 3.9 对话管理 (src/conversation_manager.py)

| 功能 | 说明 |
|------|------|
| 对话创建 | 每个用户可创建多个对话 |
| 消息存储 | 按对话存储消息历史 |
| 长期记忆 | 每 5 条消息自动提取长期记忆，用于上下文参考 |
| 对话列表 | 按时间倒序列出用户所有对话 |
| 持久化 | data/conversations/{username}_conversations.json |

### 3.10 Web 界面 (app.py)

| 功能 | 说明 |
|------|------|
| 界面风格 | ChatGPT 风格，左侧对话列表 + 右侧对话区 |
| 登录界面 | 用户名 + 密码登录 |
| 文档管理 | 文件上传/查看文档列表 |
| 知识库管理 | 构建/重载/清空知识库 |
| 对话历史 | 持久化保存，可切换对话 |

## 4. 配置项说明 (config.py)

```python
@dataclass
class Config:
    # ── DashScope ──────────────────────────────────────────────
    dashscope_api_key: str  # 必需，从环境变量读取
    llm_model: str = "qwen-plus"  # LLM 模型
    llm_base_url: str  # API 地址

    # ── Embedding ──────────────────────────────────────────────
    embedding_model: str = "text-embedding-v2"  # Embedding 模型

    # ── 文本分块 ────────────────────────────────────────────────
    chunk_size: int = 500  # 每块字符数
    chunk_overlap: int = 100  # 块间重叠字符数

    # ── 向量数据库 ─────────────────────────────────────────────
    vector_db_type: str = "memory"  # "memory" 或 "faiss"

    # ── 检索配置 ────────────────────────────────────────────────
    top_k: int = 3  # 检索返回的 top-k 个块
    use_rerank: bool = True  # 是否启用 LLM 重排序
    rerank_factor: int = 3  # Rerank 候选数量倍数
    use_query_rewrite: bool = True  # 是否启用 Query 改写
    show_citations: bool = True  # 是否在回答中标注信息来源
    show_retrieved_chunks: bool = True  # 是否打印检索到的文档块

    # ── BM25 + 向量混合检索 ─────────────────────────────────────
    use_bm25: bool = True  # 是否启用 BM25
    bm25_weight: float = 0.5  # BM25 权重 (0=纯向量, 1=纯BM25)

    # ── 用户权限 ────────────────────────────────────────────────
    default_user_role: str = "user"  # 新用户默认角色
```

## 5. 已知修复与改进

### 5.1 已修复问题

| 问题 | 修复位置 | 修复内容 | 修复版本 |
|------|---------|---------|----------|
| _doc_metadata 为 None 时增量更新失败 | `src/rag_pipeline.py` | 初始化为空 dict，所有方法检查 dict 类型 | v0.9.0 |
| 权限系统未接入问答链 | `src/rag_pipeline.py` | `answer()` 接受 `user_role` 参数并传递给检索器 | v0.9.0 |
| BM25 重复文档导致词频偏差 | `src/vector_db/bm25_retriever.py` | `add_documents()` 自动检测并跳过重复文档 | v0.9.0 |
| BM25 sources 与文档数量不匹配 | `src/vector_db/bm25_retriever.py` | build() 中自动填充默认来源值 | v0.9.0 |
| MemoryDB 单文档时 0 维数组索引错误 | `src/vector_db/memory_db.py` | 用 `flatten()` 替代 `squeeze()` | v0.9.0 |

### 5.2 当前实现评估

**优势**：

| 项目 | 说明 |
|------|------|
| ✅ 模块化设计 | 各模块职责清晰，向量数据库拆分为独立子模块 |
| ✅ 中文优化 | 使用中文优化的 Embedding 模型和分词器 |
| ✅ 混合检索 | BM25（自动去重） + 向量互补，提升检索召回率 |
| ✅ 增量更新 | 仅处理变更文档，节省 API 调用 |
| ✅ 多用户支持 | 用户注册/登录 + 权限分级 + 对话历史持久化 |
| ✅ 权限过滤 | 4 级文档权限，按用户角色过滤 |
